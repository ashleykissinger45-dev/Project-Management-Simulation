## üîÑ Workflow States

### State Definitions

**Backlog**
- Work items prioritized and ready
- Acceptance criteria defined
- Estimated with story points
- No blockers

**In Progress**
- Developer actively working
- Max 1-2 items per person (WIP limit)
- Progress updated daily
- Blockers flagged immediately

**In Review**
- Code complete, awaiting review
- Can be: code review, QA test, design review, or stakeholder approval
- Reviewer assigned
- Review started within 24 hours

**Done**
- All Definition of Done criteria met
- All acceptance criteria verified
- No open defects
- Deployed/merged
- Documentation updated
